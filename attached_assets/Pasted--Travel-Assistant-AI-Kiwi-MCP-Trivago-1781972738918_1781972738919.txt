# Техническое задание

## Название проекта

Travel Assistant AI с интеграцией Kiwi MCP и Trivago MCP

## Цель проекта

Создать AI-агента для поиска авиабилетов, отелей и комплексного подбора поездок через MCP-серверы Kiwi и Trivago.

Агент должен общаться с пользователем на русском языке, автоматически выполнять необходимые запросы к MCP-серверам и отображать журнал всех MCP-вызовов в реальном времени.

---

# Функциональные требования

## Основные возможности

### Поиск авиабилетов

Агент должен:

* принимать запросы пользователя на русском языке;
* определять города отправления и назначения;
* определять даты поездки;
* выполнять поиск через Kiwi MCP;
* отображать найденные варианты перелётов.

Примеры запросов:

* Найди билет из Berlin в Rome на следующую пятницу.
* Покажи дешёвые перелёты из Paris в Madrid на 10 июля.
* Найди билет в Barcelona на выходные.

---

### Поиск отелей

Агент должен:

* принимать запросы пользователя на русском языке;
* определять город проживания;
* определять даты заселения и выселения;
* выполнять поиск через Trivago MCP;
* отображать найденные варианты проживания.

Примеры запросов:

* Найди недорогой отель в Prague на выходные.
* Подбери отель в Vienna на 3 ночи.

---

### Комплексный подбор поездки

Агент должен уметь подбирать:

* авиабилет;
* отель;
* итоговую поездку.

Примеры запросов:

* Подбери поездку в Barcelona на 3 дня.
* Найди билет и отель для поездки в Vienna.
* Организуй поездку в Rome на выходные.

В таком режиме агент должен:

1. Сначала найти перелёт через Kiwi MCP.
2. Затем подобрать отель через Trivago MCP.
3. Вывести единый результат.

---

# Работа с русскими названиями городов

Важно:

Trivago MCP принимает только английские названия городов.

Перед обращением к Trivago MCP агент обязан:

1. Определить город из пользовательского запроса.
2. Если название указано на русском языке:

   * Москва → Moscow
   * Прага → Prague
   * Вена → Vienna
   * Барселона → Barcelona
3. Использовать английское название в запросе к Trivago.

Пользователь всегда получает ответ на русском языке.

---

# Поддержка Foursquare MCP

Дополнительно реализовать поиск достопримечательностей и интересных мест через Foursquare MCP.

Примеры запросов:

* Покажи интересные места рядом с центром Prague.

Результаты отображаются на русском языке.

# MCP-интеграции

## Kiwi MCP

Адрес:

https://mcp.kiwi.com

Алгоритм работы:

1. Вызвать initialize.
2. Сохранить mcp-session-id.
3. Использовать mcp-session-id во всех следующих запросах.
4. Выполнять поиск через инструмент search-flight.

Особенности:

* даты передавать только в формате dd/mm/yyyy;
* российские аэропорты не поддерживаются;
* ошибки MCP отображать в логах.

---

## Trivago MCP

Адрес:

https://mcp.trivago.com/mcp

Алгоритм работы:

1. Вызвать trivago-search-suggestions(query).
2. Получить:

   * id
   * ns
3. Передать id и ns в trivago-accommodation-search.
4. Вернуть результаты пользователю.

Особенности:

* использовать только английские названия городов;
* все этапы отображать в логах.

---

## Foursquare MCP

Шлюз:

POST https://gateway.pipeworx.io/foursquare/mcp

Авторизация:

Токен передаётся внутри arguments как _apiKey.

Пример вызова:

{
"jsonrpc": "2.0",
"method": "tools/call",
"params": {
"name": "search_places",
"arguments": {
"query": "museum",
"near": "Barcelona",
"limit": 5,
"_apiKey": "<FOURSQUARE_API_KEY>"
}
},
"id": 1
}

Токен хранить в переменных окружения.

# Формат ответа пользователю

Ответы всегда на русском языке.

Для перелётов выводить:

* маршрут;
* дату;
* авиакомпанию;
* цену;
* ссылку (если доступна).

Для отелей выводить:

* название;
* рейтинг;
* стоимость;
* адрес;
* ссылку.

Для комплексной поездки выводить:

* блок перелёта;
* блок проживания;
* ориентировочную общую стоимость.

Для достопримечательностей выводить:

* название;
* описание;
* адрес;
* рейтинг (если доступен).
